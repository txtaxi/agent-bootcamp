# Day04 代码带读

对应代码文件：`days/day04/http_client.py`  
对应测试文件：`tests/test_day04_http_client.py`

## 1. 今天的目标是什么

Day04 的主题是：

```python
调用外部服务时，失败了怎么办
```

这一天不是重点学某个 HTTP 库，  
而是学失败处理策略。

## 2. 核心函数

1. `should_retry`
2. `fetch_json`

## 3. `should_retry`

它的职责是判断：

```python
这次失败值不值得重试
```

当前最小规则：

- 5xx：可以考虑重试
- `timeout`、`connection`：可以考虑重试
- 4xx：通常不重试

## 4. `fetch_json`

它负责做完整请求封装：

1. 先校验输入
2. 调请求函数
3. 成功则返回统一成功结构
4. 失败则判断要不要重试
5. 最终统一返回失败结构

这说明 Day04 的重点不是“只要能请求成功”，  
而是：

```python
无论成功还是失败，都要有统一结构
```

## 5. 为什么重要

因为真实系统外部依赖是不稳定的。  
你不能默认网络永远好、服务永远正常。

## 6. 测试怎么读

测试主要覆盖：

1. 正常成功
2. timeout 后重试成功
3. 4xx 不重试
4. 5xx 重试后仍失败

## 7. 这一天真正要记什么

Day04 的一句话：

```python
请求外部服务时，失败路径必须被设计
```
