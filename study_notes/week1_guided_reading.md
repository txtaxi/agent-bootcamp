# 前7天跟读主线

## 先记一句总话

```python
前7天 = 手写一个最小 Agent
```

这 7 天不要看成 7 个零散知识点。
更好的理解方式是：

```python
输入 -> 判断 -> state -> 工具 -> 一轮执行 -> 入口
```

也就是说，前 7 天其实是在回答 4 个问题：

```python
用户说了什么？
这些信息放哪里？
系统拿什么能力去处理？
最后怎么把这一轮跑完？
```

---

## 第一层：整个系统在干嘛

你可以把前 7 天的项目想成一个很小的“办事系统”。

比如用户输入：

```python
weather in beijing
```

系统不是直接把这句话丢给模型就结束，而是要先做这些事：

1. 先接住用户输入
2. 判断这大概是什么任务
3. 把这些信息收进统一状态
4. 如果需要，就调用某个能力
5. 把结果再写回状态
6. 最后生成一个回复

这就是 Agent 的最小骨架。

所以前 7 天的核心不是语法，而是一条链：

```python
用户输入 -> 判断任务 -> 写入 state -> 调工具 -> 生成结果 -> 输出
```

---

## 第二层：每天分别在做什么

### Day01：最小路由

Day01 解决的问题是：

```python
用户到底在说什么？
```

它做的是最小的输入分类，也就是：

- 是天气问题？
- 是计算问题？
- 还是不知道？

所以 Day01 的本质是：

```python
输入处理 + 意图判断
```

你可以把它理解成门卫。
它先把用户的话做一个最初判断。

---

### Day02：统一 state

Day02 解决的问题是：

```python
这些信息放哪里？
```

这是第一周最关键的一天。

从这一天开始，系统不再把信息散着传，而是统一放进一个 `state` 字典里。
比如：

- `user_input`
- `messages`
- `intent`
- `result`
- `error`
- `thread_id`

所以 Day02 的本质是：

```python
给系统准备一个统一的信息容器
```

你现在可以直接把 Day02 看成第一周的地基。

---

### Day03：记忆 / 持久化

Day03 解决的问题是：

```python
state 能不能记住？
```

也就是说，Day02 只是把状态放进一个字典里，
Day03 开始思考：

- 这个状态能不能保存成文件？
- 下次能不能按 `thread_id` 再读回来？

所以 Day03 的本质是：

```python
状态保存与恢复
```

这意味着系统不再是“回答完就忘了”。

---

### Day04：失败别崩

Day04 解决的问题是：

```python
如果外部调用失败了怎么办？
```

真实系统里，请求失败是正常情况。
所以这一天在补“可靠性”：

- 要不要重试
- 失败后返回什么
- 怎么让错误可读
- 怎么避免系统直接崩掉

所以 Day04 的本质是：

```python
错误处理 + 健壮性
```

---

### Day05：工具层

Day05 解决的问题是：

```python
系统拿什么能力去干活？
```

这一天开始把能力封装成工具，比如：

- 天气工具
- 计算工具
- 时间工具

然后再用一个分发器统一调用。

所以 Day05 的本质是：

```python
把能力标准化成工具接口
```

你现在要开始有这种感觉：

```python
系统不是靠一个大函数包打天下，
而是靠一组标准化能力来完成任务。
```

---

### Day06：一轮执行

Day06 解决的问题是：

```python
前面的零件怎么串起来？
```

这一天第一次把前面的内容接成完整一轮：

```python
接收输入
-> 识别 intent
-> 决定工具
-> 调工具
-> 写回 state
-> 生成 reply
```

所以 Day06 的本质是：

```python
手写版 Agent 主循环 / 一轮执行
```

这一点很重要，因为你后面学 LangGraph 时，会发现它其实就是在替你组织这条流程。

---

### Day07：统一入口

Day07 解决的问题是：

```python
整个第一周怎么收口？
```

前面已经有很多模块了，
现在需要一个统一入口，把这些模块组织成一个能启动的小项目。

所以 Day07 的本质是：

```python
项目入口 / 第一周收口
```

## 第三层：第一周真正的主轴

虽然有 7 天，但第一周真正最该抓住的是这 3 个点：

### 1. Day02 的 `state`

因为这是整个系统的数据中心。

### 2. Day05 的 `tools`

因为这是系统的能力来源。

### 3. Day06 的 `run_turn`

因为这是系统第一次被真正串成完整流程。

所以第一周可以压缩成一句话：

```python
先判断用户想干什么，
再把信息放进 state，
再调用工具处理，
最后跑完一轮。
```

## 第四层：你现在不要怎么读

你现在先不要这样读：

- 逐行抠语法
- 每个函数都想背下来
- 纠结参数名为什么这么取
- 试图一次完全懂所有细节

这样很容易把自己读晕。

## 第五层：你现在应该怎么读

你先只回答这 4 个问题：

### 1. Day01 在干嘛？

```python
判断用户输入属于什么任务
```

### 2. Day02 在干嘛？

```python
把系统需要的信息统一放进 state
```

### 3. Day05 在干嘛？

```python
把能力封装成工具，并统一调用
```

```python
把输入、state、工具、回复串成完整一轮
```

如果这 4 句你能稳稳说出来，前 7 天你已经懂一半了。
